# 后续路线

## 计算机体系
再次强调一个重要概念：**抽象**。

整个计算机可以认为是从底向上的一层层抽象累积而成，每一层抽象都是对上一层抽象的简化与封装。

可以认为最普通的使用者见到的是最高抽象层，绝大部分内容已经被包装好，而面向用户裸露出键盘，鼠标，应用安装包等接口，让用户可以便捷的完成想要的操作。

对于现代计算机，没人能够完全掌握每个抽象层，这些抽象层存在的意义也是给人们合力构建计算机时提供标准与便利。对于未来大致的学习方向，也可以说是对自己喜欢的抽象层进行深入学习，而对其他抽象层有基本的了解，来方便上下接口的对接。

*以下按逻辑顺序，而不是学习顺序说明*

计算机是逻辑机器，由底层最基础的门电路保证运行的正确性。再向上有CPU，内存等不同硬件，相互配合完成高抽象层发出的指令。计算机的底层部分被称为**体系结构**。

对与大部分人，需要了解的是CPU，内存等的运行逻辑，由于底层决定上层，许多上层的问题需要到硬件层寻找答案，因此基础的了解还是有必要的，对应课程为计算机组成原理，**CSAPP**这本书覆盖了大部分程序员需要知道的硬件内容。

再往上就是操作系统，目前主要有Windows，Linux的各种发行版和UNIX。操作系统是硬件到软件中极为重要的一环。操作系统一方面协调底层各种复杂的硬件，一方面为上层运行的程序提供极为丰富的接口来调用资源，因此操作系统本身也是极为复杂的。由于Windows的源码不开源而且机制复杂，推荐用Linux来学习OS，其中发行版Ubuntu有很友好的图形界面，能减轻初见命令行的不适应，利用WSL（Windows Subsystem for Linux）学习也很方便。南京大学的蒋炎岩老师讲的[操作系统](https://csdiy.wiki/%E6%93%8D%E4%BD%9C%E7%B3%BB%E7%BB%9F/NJUOS/)很好，非常推荐。

再往上就是我们所见的各种软件了。被称为“魔法书”的SICP是MIT的大一教材，专注于探讨如何构建软件和系统。初版SICP使用Lisp的方言Scheme，~~你们可能听见这个偏门语言就不想学了。~~ 我正在看的[Javascript版的SICP](https://sourceacademy.org/) 目前好像还没有中文版，因此最推荐 [CS61A](https://csdiy.wiki/%E7%BC%96%E7%A8%8B%E5%85%A5%E9%97%A8/CS61A/) 作为入门课。该课程使用 Python 版的SICP作为教科书，链接里也有该书的中文资源。这门课对初学者有点难，但值得花时间和精力学习，你将对**编程本身**有更高水平的认识。

最终不同作用的软件系统分化为不同的二级学科，如编译原理、数据库、数据结构与算法等，学校中一般会为每部分都开设一门课，在一个学期的时间中学习。这也意味这学校的课无法在每一个方向都进行深入，重在全而不在深，如果对某个二级学科感兴趣，就需要自己花费额外的时间去学习（特定方向的社团、[CS自学指南](https://csdiy.wiki/)上更深入的课程等）。

## 关于算法竞赛
我在入门部分提到，大部分人在入门时并没有使用编程语言这一**工具**的**目的**，而算法竞赛对于入门者的意义就是，它可以提供给你**目的**。

算法本质上关注的是数学问题，但是不同点在于，传统的数学关注“是什么”的定义，而算法关注“怎么做”的过程，这一过程需要借助编程语言进行表达。

初级的算法题目，考察内容基本在学校的离散数学、数据结构、算法等课程之内，对后续课程乃至找工作面试时都会有帮助。同时题目更具有趣味性，通过题目（AC）时也有很好的正反馈，所以很推荐大一的同学去学一些算竞的基本内容，同时锻炼自己的代码能力。当学习到更深入的领域，则根据自己情况选择要不要继续打下去。算竞是一种选择，至少在无副作用的范围内都可以试试。
